# 実装ルール

## 全体方針
- 画面は薄く、ロジックは hooks / services に寄せる
- 出題ロジックは1本化する
- 保存は storage service 経由に寄せる
- UI と計算ロジックを混ぜすぎない

## React / TypeScript 方針
- TypeScript の型を明示する
- `any` は避ける
- state に入れる値は必要最小限にする
- derived data は可能な限り計算で出す
- localStorage 直叩きを各画面に散らさない

## ページ責務
- TopPage: 演算選択、開始、設定/成績への導線
- SettingsPage: モード、時間、難易度、特別ルール
- PlayPage: プレイ進行
- ResultPage: 1プレイ直後の結果
- ScorePage: 累積成績

## 後方互換より優先すること
今回は新規構築なので、旧版互換のための複雑な実装は不要。
ただし将来拡張しやすい構造は意識すること。

## UI 方針
- スマホ縦持ち最優先
- ただし Play 画面だけは将来的に横長2カラムへ対応可能な構造にする
- 正解時は進行を止めない
- 不正解時のみフィードバック表示

## 保存データ方針
- 軽量優先
- 問題文文字列は保存しない
- 最新1プレイ分だけ詳細を保持
- 集計とベストはキャッシュする
